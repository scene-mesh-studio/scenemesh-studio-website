## 💻 开发问题

### Q: 如何定义实体关系？

**A:** 使用装饰器定义关系：

```typescript
@Entity()
class User {
  @OneToMany(() => Post, post => post.author)
  posts: Post[];
}

@Entity()
class Post {
  @ManyToOne(() => User, user => user.posts)
  author: User;
  
  @ManyToMany(() => Tag)
  tags: Tag[];
}
```

### Q: 如何实现数据验证？

**A:** 使用内置验证器：

```typescript
@Entity()
class User {
  @Field({
    type: 'string',
    validators: [
      { type: 'required' },
      { type: 'email' },
      { type: 'minLength', value: 5 }
    ]
  })
  email: string;
  
  @Field({
    type: 'number',
    validators: [
      { type: 'min', value: 18 },
      { type: 'max', value: 120 }
    ]
  })
  age: number;
}
```

### Q: 如何处理异步操作？

**A:** Entity Engine 全面支持 async/await：

```typescript
async function createUser(data: CreateUserData) {
  const entityManager = getEntityManager();
  
  // 开启事务
  return await entityManager.transaction(async (em) => {
    // 创建用户
    const user = await em.create('User', data);
    
    // 发送欢迎邮件
    await sendWelcomeEmail(user.email);
    
    return user;
  });
}
```

### Q: 如何优化查询性能？

**A:** 几种优化方式：

```typescript
// 1. 使用预加载避免 N+1 问题
const users = await entityManager.find('User', {
  include: ['posts', 'profile']
});

// 2. 使用查询缓存
const users = await entityManager.find('User', {
  cache: { ttl: 300 } // 缓存5分钟
});

// 3. 使用索引
@Entity()
class User {
  @Field({ index: true })
  email: string;
  
  @Field({ index: 'compound', with: ['status'] })
  createdAt: Date;
}
```

### Q: 如何实现实时数据同步？

**A:** 使用内置的实时系统：

```typescript
// 服务端
const realTimeManager = getRealTimeManager();

// 订阅实体变更
realTimeManager.subscribe('User', {
  events: ['created', 'updated', 'deleted'],
  filter: (event) => event.entity.isPublic
});

// 客户端
const client = new EntityEngineClient();
client.subscribe('User', {
  onUpdate: (user) => updateUI(user),
  onDelete: (userId) => removeFromUI(userId)
});
```

## ⚡ 性能问题

### Q: 为什么查询很慢？

**A:** 常见原因和解决方案：

1. **缺少索引**
   ```typescript
   // 添加索引
   @Field({ index: true })
   userId: string;
   ```

2. **N+1 查询问题**
   ```typescript
   // 使用预加载
   const posts = await em.find('Post', {
     include: ['author', 'comments']
   });
   ```

3. **查询过于复杂**
   ```typescript
   // 使用原生查询
   const result = await em.query(`
     SELECT p.*, u.name as author_name 
     FROM posts p 
     JOIN users u ON p.user_id = u.id 
     WHERE p.status = 'published'
   `);
   ```

### Q: 内存使用量过高怎么办？

**A:** 优化内存使用：

```typescript
// 1. 启用查询流式处理
const users = await em.findStream('User', {
  batchSize: 1000,
  onBatch: (batch) => processBatch(batch)
});

// 2. 清理未使用的缓存
em.cache.clear('User');

// 3. 限制查询结果数量
const users = await em.find('User', {
  limit: 100,
  select: ['id', 'name'] // 只选择必要字段
});
```

### Q: 并发处理能力不足？

**A:** 提高并发性能：

```typescript
// 1. 调整连接池大小
const entityEngine = new EntityEngine({
  database: {
    pool: {
      min: 10,
      max: 100,
      acquireTimeoutMillis: 30000
    }
  }
});

// 2. 使用并发控制
import { queue } from 'async';

const processingQueue = queue(async (task) => {
  return await processEntity(task);
}, 10); // 并发数限制为 10

// 3. 启用查询缓存
const result = await em.find('User', {
  cache: { ttl: 300, key: 'active_users' }
});
```

## 🔧 故障排除

### Q: "连接数据库失败" 错误？

**A:** 检查以下几点：

1. **数据库服务是否运行**
   ```bash
   # PostgreSQL
   sudo systemctl status postgresql
   
   # MySQL
   sudo systemctl status mysql
   ```

2. **连接配置是否正确**
   ```typescript
   const config = {
     host: 'localhost',
     port: 5432,
     database: 'myapp',
     username: 'user',
     password: 'password'
   };
   ```

3. **网络连接是否正常**
   ```bash
   # 测试连接
   telnet localhost 5432
   ```

### Q: "实体未找到" 错误？

**A:** 可能的原因：

1. **实体未注册**
   ```typescript
   // 确保实体已注册
   EntityEngine.register([User, Post, Comment]);
   ```

2. **导入路径错误**
   ```typescript
   // 检查导入路径
   import { User } from './entities/User';
   ```

3. **实体名称不匹配**
   ```typescript
   // 确保名称一致
   @Entity('users') // 表名
   class User {
     // ...
   }
   ```

### Q: "权限被拒绝" 错误？

**A:** 权限问题排查：

1. **检查用户权限**
   ```typescript
   const hasPermission = await accessControl.check(
     user, 
     resource, 
     'read'
   );
   ```

2. **验证角色分配**
   ```typescript
   const userRoles = await roleManager.getUserRoles(userId);
   console.log('User roles:', userRoles);
   ```

3. **检查策略配置**
   ```typescript
   const policies = await policyStore.findApplicable({
     user: user,
     resource: resource,
     action: 'read'
   });
   ```

### Q: 如何处理错误？

**A:** 统一错误处理：

```typescript
// 自定义错误类
class BusinessError extends Error {
  constructor(
    message: string,
    public code: string,
    public statusCode: number = 400
  ) {
    super(message);
    this.name = 'BusinessError';
  }
}

// 全局错误处理中间件
app.use((error, req, res, next) => {
  if (error instanceof BusinessError) {
    return res.status(error.statusCode).json({
      error: error.message,
      code: error.code
    });
  }
  
  // 记录未知错误
  console.error(error);
  res.status(500).json({ error: 'Internal server error' });
});
```

### Q: 如何编写测试？

**A:** 完整的测试策略：

```typescript
// 单元测试
describe('UserService', () => {
  let userService: UserService;
  let mockEntityManager: jest.Mocked<IEntityManager>;

  beforeEach(() => {
    mockEntityManager = createMockEntityManager();
    userService = new UserService(mockEntityManager);
  });

  it('should create user successfully', async () => {
    const userData = { name: 'John', email: 'john@example.com' };

    expect(mockEntityManager.create).toHaveBeenCalledWith('User', userData);
  });
});
```
